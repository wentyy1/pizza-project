
# Roadmap розробки pizzaDay

## Тиждень 1-2: Базова функціональність
- [x] Налаштування React frontend
- [x] Підключення PostgreSQL бази даних
- [x] Базова структура бекенду на Node.js/Express
- [x] Міграції бази даних для таблиць pizzas та categories

## Тиждень 3: API для меню піц
- [ ] Реалізація endpoints для роботи з піцами:
  - `GET /api/pizzas` - отримання всіх піц
  - `GET /api/pizzas/:id` - отримання конкретної піци
  - `GET /api/categories` - отримання категорій
- [ ] Інтеграція API з React frontend
- [ ] Компонент відображення меню піц

## Тиждень 4: Система авторизації
- [ ] Модель користувачів в PostgreSQL
- [ ] Endpoints для реєстрації та логіну:
  - `POST /api/auth/register` - реєстрація
  - `POST /api/auth/login` - авторизація
- [ ] JWT токени для безпеки
- [ ] Захист маршрутів на фронтенді

## Тиждень 5: Кошик та замовлення
- [ ] CRUD операції для кошика:
  - `POST /api/cart/add` - додати піцу
  - `GET /api/cart` - переглянути кошик
  - `DELETE /api/cart/:id` - видалити з кошика
- [ ] Модель замовлень в базі даних
- [ ] Endpoint для створення замовлення

## Тиждень 6: Адмін-панель
- [ ] CRUD для адміністратора:
  - `POST /api/admin/pizzas` - додати піцу
  - `PUT /api/admin/pizzas/:id` - редагувати піцу
  - `DELETE /api/admin/pizzas/:id` - видалити піцу
- [ ] Компонент адмін-панелі на React
- [ ] Захист адмін-маршрутів

## Майбутні функції (після базової реалізації)
- Система відстеження статусу замовлень
- Інтеграція платіжних систем
- Push-сповіщення
- Система знижок та промокодів
- Мобільний додаток
