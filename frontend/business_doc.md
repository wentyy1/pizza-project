# Бізнес-документ системи pizzaDay

## Назва системи та опис
**pizzaDay** - це веб-додаток для замовлення піци з повним циклом обробки замовлень. Система дозволяє користувачам переглядати меню, складати замовлення, відстежувати статус доставки та управляти своїми профілями.

Система інтегрує фронтенд на React з бекендом на Node.js та базою даних PostgreSQL, забезпечуючи швидке та зручне замовлення піци. Користувачі можуть легко знаходити улюблені страви, налаштовувати складові піци та відстежувати історію своїх замовлень.

## Користувачі (ролі)
- **Клієнт** - особа, яка робить замовлення піци
- **Адміністратор** - управляє меню, переглядає всі замовлення
- **Кур'єр** - відповідає за доставку замовлень (майбутня роль)

## Основні сценарії використання
1. **Реєстрація та авторизація** - користувач створює обліковий запис або входить у систему
2. **Перегляд меню** - клієнт переглядає доступні піци з фільтрацією за категоріями
3. **Створення замовлення** - клієнт додає піци до кошика, оформлює та оплачує замовлення
4. **Відстеження замовлення** - клієнт бачить статус свого замовлення в реальному часі
5. **Управління меню** - адміністратор додає, редагує або видаляє позиції з меню

## Функціональні вимоги
- Реєстрація та авторизація користувачів
- CRUD операції для меню піц
- Система кошика для формування замовлень
- Історія замовлень для кожного користувача
- Пошук та фільтрація піц за категоріями
- Адмін-панель для управління контентом

## Нефункціональні вимоги
- **Швидкодія**: Завантаження сторінок < 3 секунд
- **Безпека**: Хешування паролів, захист від SQL-ін'єкцій
- **Зручність**: Адаптивний інтерфейс для мобільних пристроїв
- **Надійність**: Резервне копіювання бази даних щодня

## Обмеження
- Система не підтримує онлайн-оплату (на першому етапі)
- Не має інтеграції з картами для відстеження кур'єрів
- Не підтримує push-сповіщення
- Не має системи знижок та промокодів (на першому етапі)
